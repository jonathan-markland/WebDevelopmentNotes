overflow

The overflow CSS shorthand property sets the desired behavior for an element's overflow — i.e. when an element's content is too big to fit in its block formatting context — in both directions.

Link to overflow on MDN
  • overflow: visible;
  • overflow: hidden;
  • overflow: scroll;
  • overflow: auto;

position

The position CSS property sets how an element is positioned in a document. The top, right, bottom, and left properties determine the final location of positioned elements.

Link to position on MDN
  • position: static;
  • position: relative; top: 40px; left: 40px;
  • position: absolute; top: 40px; left: 40px;

The static setting positions the div in the natural place in the original flow layout.

The relative setting reserves space in the original flow layout while moving the div relative to where it would be if static.

The absolute setting takes no space in the original flow layout, and positions relative to most recent positioned parent.

Both relative and absolute start an new stacking context when z-index is not auto.

If you are trying to hide absolute positioned elements make sure the container of those absolute positioned elements is relatively positioned.

display

The display CSS property sets whether an element is treated as a block or inline element and the layout used for its children, such as flow layout, grid or flex.

Formally, the display property sets an element's inner and outer display types. The outer type sets an element's participation in flow layout; the inner type sets the layout of children.

Link to display on MDN
  • display: inline;
  • display: block;
  • display: inline-block;
  • display: flex;
  • display: inline-flex;
  • display: none;

NB: There is a new syntax omitting the hyphen, designed to emphasise two independent mode settings (inner and outer).
